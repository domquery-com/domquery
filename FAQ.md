# Frequently Asked Questions (FAQ)

## General Questions

### Q: Is DomQuery compatible with other DOM manipulation libraries?
**A:** DomQuery provides similar API patterns but is not fully compatible. Some modifications may be required when migrating existing code to DomQuery.

### Q: Which browsers does DomQuery support?
**A:** DomQuery supports all modern browsers. IE support is limited. For more details, please refer to the [official website](https://domquery.com/?introduction).

### Q: Do I need npm to use DomQuery?
**A:** No. You can use npm installation, CDN, or direct download. For more details, please refer to [README.md](README.md).

## Technical Questions

### Q: Does history management work automatically in WebView environments?
**A:** Yes, DomQuery Alert automatically manages history by default. You can disable it by setting `History: false`.

### Q: Does Ajax work with file:// URLs?
**A:** Yes, DomQuery Ajax automatically handles file:// URLs. WebView Bridge is also automatically supported.

### Q: 681 options seem too many. Do I need to use all of them?
**A:** No. Most options are optional, and you can use the library with default values. Use only the options you need.
